1 # ESP32 U8G2 library support
2 There is an excellent open source library called `u8g2` that can be found on Github here:
3 
4 [https://github.com/olikraus/u8g2](https://github.com/olikraus/u8g2)
5 
6 The purpose of the library is to provide a display independent driver layer for monochrome displays including LCD and OLED.
7 The library "knows" how to driver the underlying displays as well as providing drawing primitives including text, fonts, lines and
8 other geometrical shapes.
9 
10 A version of the library is available in cleanly compiling C and compiles without incident on the ESP32-IDF framework.
11 
12 However, since the library is agnostic of MCU environments and will work on a variety of boards, there has to be a mapping from
13 the functions expected by the library to the underlying MCU board hardware. This includes driving GPIOs, I2C, SPI and more.
14 
15 The code in this folder provides a mapping from U8g2 to the ESP32 ESP-IDF. This should be included in your build of U8g2 applications.
16 
17 To use with the ESP32, we must invoke the `u8g2_esp32_hal_init()` function before invoking any of the normal U8g2 functons. What
18 this call does is tell the ESP32 what pins we wish to map. Here is an example of SPI use:
19 
20 ```
21 u8g2_esp32_hal_t u8g2_esp32_hal = U8G2_ESP32_HAL_DEFAULT;
22 u8g2_esp32_hal.clk = PIN_CLK;
23 u8g2_esp32_hal.mosi = PIN_MOSI;
24 u8g2_esp32_hal.cs = PIN_CS;
25 u8g2_esp32_hal.dc = PIN_DC;
26 u8g2_esp32_hal.reset = PIN_RESET;
27 u8g2_esp32_hal_init(u8g2_esp32_hal);
28 ```
29 
30 and here is an example of I2C use:
31 
32 ```
33 u8g2_esp32_hal_t u8g2_esp32_hal = U8G2_ESP32_HAL_DEFAULT;
34 u8g2_esp32_hal.sda = PIN_SDA;
35 u8g2_esp32_hal.scl = PIN_SCL;
36 u8g2_esp32_hal_init(u8g2_esp32_hal);
37 ```
38 
39 Note: For I2C, after calling the display specific setup, invoke:
40 
41 ```
42 u8x8_SetI2CAddress(&u8g2.u8x8,<address>);
43 ```
44 
45 Note that `<address>` is the I2C address already shifted left to include the read/write flag. For example, if you are using an SSD1305, instead of supplying the address `0x3C` which would be `0011 1100` supply `0x78` which would be `0111 1000`.
46 
47 The function takes as input a `u8g2_esp32_hal` structure instance which has the logical pins that U8g2 needs mapped to the
48 physical pins on the ESP32 that we wish to use. If we don't use a specific pin for our specific display, set the value to
49 be `U8G2_ESP32_HAL_UNDEFINED` which is the default initialization value.
50 
51 Remember, ESP32 pins are not hard-coded to functions and as such, all the GPIO pins on the ESP32 are open for use. Following
52 this initialization, we can use U8g2 as normal and described in the U8g2 documentation.
53 
54 ## Compiling U8G2
55 To use the actual U8g2 library in your ESP32 project, perform the following steps:
56 
57 1. Create a directory called `components` in your main project directory.
58 2. Change into the `components` directory.
59 3. Run `git clone https://github.com/olikraus/u8g2.git` to bring in a the latest copy of u8g2 library.
60 4. Change into the `u8g2` directory.
61 5. Create a file called `component.mk`
62 6. Enter the following in the `component.mk` file:
63 ```
64 COMPONENT_SRCDIRS:=csrc
65 COMPONENT_ADD_INCLUDEDIRS:=csrc
66 ```

69 ## Development
70 While in principal, there should be nothing specific needed beyond this addition to make U8g2 work on the ESP32, only a small
71 number of boards have been tested.
