5 ---
6 This posts data exactly as specified with no extra processing whatsoever.
7 
8 If you start the data with the letter @, the rest should be a filename. Data
9 is posted in a similar manner as --data does, except that newlines and
10 carriage returns are preserved and conversions are never done.
11 
12 Like --data the default content-type sent to the server is
13 application/x-www-form-urlencoded. If you want the data to be treated as
14 arbitrary binary data by the server then set the content-type to octet-stream:
15 -H "Content-Type: application/octet-stream".
16 
17 If this option is used several times, the ones following the first will append
18 data as described in --data.
