1 # ESP32 FTP Server
2 The world is awash in excellent FTP servers both as stand-alone applications as well as libraries that can be linked with to build ones own integrated FTP server. This project is yet another.
3 
4 The design intent is that it will be available to ESP32 based applications to server files from a Posix file system.
5 
6 ## FTP Protocol
7 We will be concentrating exclsuively on the FTP Server protocol. This will allow an ESP32 to serve up files to FTP clients and allow FTP clients to deposit new files.
8 
9 The FTP Protocol is described in [RFC 959](https://tools.ietf.org/html/rfc959).
10 
11 Our server engine will implement the Server-PI (Server Protocol Interpreter) and the Server-DTP (Server Data Transfer Process).
12 
13 ## Commands
14 
15 ### PASV
16 The client has requested that the server enter passive mode. The response must be:
17 
18 ```
19 227 Entering Passive Mode (h1,h2,h3,h4,p1,p2).
20 ```
21 
22 ### PORT
23 Client requests that the server form a data connection to the given host and port. This is used in FTP *active* mode.
24 ```
25 PORT {h1,h2,h3,h4,p1,p2}
26 ```
27 port number is p1*256 + p2
28 
29 ### RETR
30 This command represents a request *from the client* to retrieve a file held by the server.
31 
32 ### STOR
33 This command represents a request *from the client* to stor a file on the server.
34 
35 ### SYST
36 Determine the System type of the FTP server.
37 ```
38 SYST
39 ```
40 The recommended response is:
41 ```
42 215 UNIX Type: L8
43 ```
44 
45 ### TYPE
46 Define the type of data to be transmitted. Codes include:
47 
48 * `A` - ASCII
49 * `E` - EBCDIC
50 * `I` - Image (binary)
51 
52 Example:
53 ```
54 TYPE I
55 ```
56 
57 ### USER
58 Supply the user name that the client wishes to present to the FTP server.
59 ```
60 USER {name}
61 ```
62 
63 A response of `331` will request a password and should then be followed by the `PASS` command.
64 
65 ## Active vs Passive
66 The FTP server can communicate data to the client in one of two modes called *active* and *passive*. In active mode, the FTP client is responsible for setting up a listening socket. When the FTP client is ready to receive data, it sends a `PORT` command informing the server of the IP address and port against which the FTP server should form the connection.
67 In passive mode, the FTP client sends a `PASV` command to the FTP server. The FTP server then responds with a host and port pair which will be listened upon to receive a connection request *from* the client. Once received, the data can then flow over this connection.
